# 第二阶段优化规划

## 目标

第二阶段继续做低风险、可验证的稳定性优化，优先减少构建警告和潜在空引用，不改变现有 UI 交互。

## 执行顺序

### 1. 修正 `RelayCommand` 可空性

问题：

- `ICommand.CanExecute` 和 `ICommand.Execute` 的参数应允许 `object?`。
- `CanExecuteChanged` 事件应允许可空处理器。
- `canExecute` 可为空，但字段类型当前不是可空。

验收：

- `RelayCommand.cs` 不再产生可空性接口实现警告。

### 2. 加固 `App.xaml.cs` 启动和注册表逻辑

问题：

- `notifyIcon` 和 `_configManager` 在启动前为空，但字段没有表达可空。
- 查找托盘菜单项时可能返回空。
- 注册表 `OpenSubKey` 可能返回空。
- DPI 计算中 `MainWindow` 可能为空。

验收：

- 注册表写入和删除在 key 获取失败时有明确提示或安全返回。
- 托盘菜单项为空时不会直接空引用。
- 应用退出时释放资源仍然正常。

### 3. 更新检查配置常量化

问题：

- 更新接口地址和超时时间写在方法内部。
- 反序列化结果的 URL、文件名没有空值保护。

优化方式：

- 将更新接口基地址、超时时间、应用名等提取为常量。
- 拼接 URL 时使用 `Uri.EscapeDataString`。
- 启动更新程序前校验 URL 和文件名。

验收：

- `CheckForUpdates` 逻辑更清晰。
- 更新响应缺字段时不会继续启动 updater。

### 4. 构建验证

执行：

```powershell
dotnet build
```

验收：

- 构建成功，0 个错误。
- 记录剩余警告类型，作为第三阶段输入。

## 本阶段不处理

- 托盘库 `NU1701` 替换。这需要选择新依赖并做运行验证，风险高于本阶段目标。
- 大规模拆分 `App.xaml.cs`。本阶段只做防御性加固，后续再拆服务。
- 全量清理所有可空性警告。优先处理本阶段触及的文件。
